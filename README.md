# An HTML5 Text-Based Story Game
This is an interactive text-based storytelling game; this experience was developed in coordination with a relative of mine who enjoys writing. To develop this experience, I've utilized JavaScript, Hyper-Text Markup Language (HTML), Cascading Style Sheets (CSS), and JQuery to deliver a game that is as immersive as a text-based game can get. To enjoy it to its highest potential, let yourself be immersed in the story. Turn every prompt into an opportunity to predict the future. Your choices decide your experience.

<h2>Project Summary</h2>
I now consider this project complete. The source code works fine for one of my old projects. Although I would make changes with my current programming knowledge, it would require a full overhaul of the JS files. I'm not the type who constantly revisits really old projects. I do what needs to be done and move on to the next project with the learning outcomes of the last project in mind. I spent countless hours and weeks on each episode. It isn't unrealistic to say that I've spent more than 150 hours on both episodes combined during their development periods. For Episode One, I spent one week, from ~11 am in the morning to midnight every day. Considering lunch and dinner breaks, that's ~11 hours a day. With the same schedule, I worked on Episode Two for two weeks.
<br>
<br>
One has to keep in mind that I was playing the role of a writer, editor, developer, sound designer, and project manager. Juggling all of these on my own with minimal input from the original writer was time-consuming. I had to take the story given to me by the original writer and turn it into a multi-faceted story, changing the story depending on what choices the user makes, while still maintaining the logic and direction of the original story. I also had to create a rather lengthy storyboard to keep track of every branch of the story, so that I didn't get confused during development. After that, I developed a sort of code-based game engine that allowed me to handle sound, prompts, bug fixing, and text displaying very easily. I packed the entire story into a JavaScript array of objects, setting each object as a scene. Then I used a function in the game engine to run through each object and display its contents, as well as prompt the user for input on the necessary scenes.
<br>
<br>
<strong>Overall</strong>, the experience, however stressful, was enjoyable at the end. I'm proud of the work I was able to put in and the web app that I was able to develop. I love staying in the development mindset, and this project allowed me to do that for extended periods of time. I hope one day I get to work on another similar project that allows me to really put my developer brain to use on something creative.
